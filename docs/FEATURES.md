# 小龟寻物现阶段功能文档

更新日期：2026-06-18

## 1. 产品定位

小龟寻物是一款个人用 Android App，用来记录“东西放在哪里”。

典型场景：

- 临时把证件、转接头、钥匙、卡片、票据放到某个地方，怕之后忘记。
- 打开 App 拍一张照片，照片只进入小龟寻物，不混入系统相册。
- 之后用自然语言搜索，比如“黑色转接头在哪”“医保卡放哪了”。
- App 根据照片识别结果、用户备注、时间和 GPS 信息，返回可能的位置和当时照片。

## 2. 当前已实现功能

### 2.1 首页拍照

- 首页是相机预览和拍照按钮。
- 点击拍照后，会先把照片保存到 App 私有目录。
- 照片不会写入系统相册。
- 保存后会创建一条“记忆记录”，初始状态为待识别。
- 拍照后会自动启动识别流程。
- 如果相机还没准备好，会提示检查相机权限。
- 如果拍照失败，会显示失败提示，不会静默无反应。

### 2.2 照片与缩略图保存

- 原图保存到 App 私有目录下的 `memory_photos`。
- 缩略图保存到 App 私有目录下的 `memory_thumbnails`。
- 数据库只保存文件路径，不把图片放进数据库。

### 2.3 GPS 位置记录

- 拍照时会尝试读取 GPS。
- GPS 读取不会阻塞照片入库。
- 如果 GPS 权限未给、定位服务关闭、定位超时或定位失败，照片仍会保存。
- GPS 成功时，会记录纬度、经度和精度。

### 2.4 MiMo 多模态识别

- 使用 MiMo `mimo-v2.5`。
- 识别请求会把照片作为图片输入发给模型。
- 识别结果会结构化保存，包括：
  - 主要物品
  - 物品别名
  - 场景描述
  - 位置判断
  - 搜索摘要
  - 置信度
- 没有 API Key、网络失败或模型返回异常时，照片不会丢。
- 失败记录会保留为可重试状态。

### 2.5 MiMo API Key 设置

- 设置页可以填写 MiMo API Key。
- API Key 保存在系统安全存储中，不写入源码。
- 保存前会校验 API Key 格式。
- 如果误填了中文说明、空格、换行等内容，会提示格式错误，不会继续保存。
- 设置页提供“测试 MiMo 连接”按钮。
- 连通性测试使用内置测试图片走多模态接口，不是纯文本测试。

### 2.6 识别进度页

首页左上角状态按钮可进入识别进度页。

当前可查看：

- 每张照片的缩略图。
- 拍摄时间。
- 识别状态：
  - 待识别
  - 识别中
  - 已完成
  - 失败
- 错误提示。
- 已识别出的主要物品、别名、场景、位置判断、搜索摘要和置信度。

当前可操作：

- 单张手动识别。
- 一键识别全部待处理照片。
- 手动刷新列表。

### 2.7 自然语言搜索

- 首页底部有搜索输入框。
- 用户可以输入自然语言问题。
- App 会读取已识别的记忆记录摘要。
- 由于当前定位是个人轻量使用，默认把已识别记录摘要一起交给 MiMo 做模糊匹配。
- 搜索结果页会展示：
  - 模型回答
  - 匹配照片
  - 匹配理由
  - 拍摄时间
  - 用户位置备注
  - AI 位置推测
  - GPS 信息

### 2.8 Android 应用身份

- 应用显示名：小龟寻物。
- 包名：`app.xiaogui.xunwu`。
- 包名不包含个人姓名。
- 已接入自定义应用图标。
- 支持普通 launcher icon 和 adaptive icon。

### 2.9 测试 APK 输出


输出位置：

```text

```

## 3. 数据状态说明

每条记忆记录会保存一个状态：

- `pending`：照片已保存，等待识别。
- `recognizing`：正在调用 MiMo 识别。
- `recognized`：识别成功，可以被搜索。
- `failed`：识别失败，可以手动重试。

记录中保存的信息包括：

- 记录 ID
- 原图路径
- 缩略图路径
- 拍摄时间
- GPS 信息
- 用户备注位置
- AI 识别字段
- 状态
- 错误信息
- 创建时间
- 更新时间

## 4. 当前容错策略

- 拍照优先保存照片，后续步骤失败不影响照片入库。
- GPS 失败不影响保存和识别。
- 没有 API Key 时，记录保持待识别或失败状态，可后续补 Key 再识别。
- MiMo 请求失败时，错误会保存到记录里，用户可在识别进度页查看并重试。
- API Key 格式错误会提前拦截，避免底层 HTTP Header 错误直接暴露给用户。

## 5. 当前已知限制

- 目前没有照片删除、批量删除或编辑记录页面。
- 目前没有拍照后的补充备注输入流程。
- 当前搜索只查已识别成功的记录。
- 当前没有后台任务调度；App 启动时会尝试处理积压识别，识别页也可手动触发。
- 当前没有真正的进度百分比，只有记录级状态。
- 当前没有地图展示 GPS 位置。
- 当前没有导出、备份或跨设备同步。
- 当前没有正式 release 签名包，只有 debug APK。

## 6. 建议下一步

优先级较高：

- 拍照后提供一个轻量备注入口，但不阻塞自动入库和识别。
- 识别进度页增加删除记录功能。
- 搜索结果页支持从结果跳到完整记录详情。
- 首页显示最近几条记忆，方便确认刚拍的照片已经入库。

优先级中等：

- API Key 设置页增加“清除 Key”按钮。
- 识别失败时提供更细分的错误类型。
- 记录详情页支持编辑用户备注。
- 增加真正后台重试任务。

优先级较低：

- 地图查看 GPS。
- 数据导出/备份。
- 正式 release 签名和版本号管理。
